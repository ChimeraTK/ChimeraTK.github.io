Return current calibration status as StepperMotorCalibrationStatus object.
Currently following status are forseen:
1) StepperMotorCalibrationStatusType::M_CALIBRATION_UNKNOWN - calibration status in unknown. This is default value after object creation.
2) StepperMotorCalibrationStatusType::M_CALIBRATED - motor calibrated.

7) StepperMotorCalibrationStatusType::M_CALIBRATION_NOT_AVAILABLE - motor calibration is not possible because motor doesn't have physical end switches.

StepperMotorStatusAndError mtca4u::StepperMotor::getStatusAndError ( )

Return object which contains current status and error.

Returns
StepperMotorStatusAndError - object which contains of status and error code

Return current motor status and error as StepperMotorStatusAndError object.
It give possibility to check what was an error type when status is set to StepperMotorStatusTypes::M_ERROR. Status and error create pair which describes current state of the object. Every time it is called it checks status of the motor.

Currently following status are forseen:
1) StepperMotorStatusTypes::M_OK - motor is OK and ready for new commands.
2) StepperMotorStatusTypes::M_DISABLED - motor is disabled. No action can be done. to change this use 'void setEnabled(bool enable)' method.
3) StepperMotorStatusTypes::M_IN_MOVE -motor in in move.
4) StepperMotorStatusTypes::M_NOT_IN_POSITION - target position in different than current one and motor is not moving
5) StepperMotorStatusTypes::M_SOFT_POSITIVE_END_SWITCHED_ON - target position in greater than value set by 'void setMaxPositionLimit(float maxPos)' method.
6) StepperMotorStatusTypes::M_SOFT_NEGATIVE_END_SWITCHED_ON - target position in smaller than value set by 'void setMinPositionLimit(float maxPos)' method.
7) StepperMotorStatusTypes::M_ERROR - motor is in error state. For details about error check 'StepperMotorError getError()' method.
Currently following errors are forseen:
1) StepperMotorErrorTypes::M_NO_ERROR - motor is OK and ready for new commands.
2) StepperMotorErrorTypes::M_CONFIG_ERROR_MIN_POS_GRATER_EQUAL_TO_MAX - error in configuration of software position limits
3) StepperMotorErrorTypes::M_COMMUNICATION_LOST - errors and problems with communication with firmware
4) StepperMotorErrorTypes::M_NO_REACION_ON_COMMAND - hardware is no reacting on command.
5) StepperMotorErrorTypes::M_HARDWARE_NOT_CONNECTED - hardware is not connected.

void mtca4u::StepperMotor::setTargetPosition ( float  newPosition)

Set new position for the motor (no blocking)

Parameters
newPosition- new position for the motor
Returns
void

Motor behavior depends on the Autostart flag value.
When Autostart flag is set to TRUE motor will automatically start moving to desired position by calling 'void start()' method.
When Autostart flag is set to FALSE motor will NOT automatically start moving to desired position. Additionally, execution of start() method is necessary to start motor.
This is NO blocking method so user needs to monitor motor status.
Position is expressed in arbitrary units ex. mm or fs or deg or steps. Recalculation from arbitrary unit to steps is done by object of StepperMotorUnitsConverter.
StepperMotorUnitsConverter can be set by setStepperMotorUnitsConverter method. When not set conversion coefficient is 1 (steps = 1 x units).

void mtca4u::StepperMotor::stop ( )
virtual

Stop the motor.

Function stop the motor in a way that real position from the motor is set as a target position.
It also interrupts all blocking functions as 'StepperMotorStatusAndError moveToPosition(float newPosition)'.
Delays in communication can trigger effect that motor will move a few steps in current movement direction and later will move couple steps back, but this issue can be filtered by motor hysteresis effect.
