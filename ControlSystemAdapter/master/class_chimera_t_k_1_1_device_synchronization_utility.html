Detailed Description

Utility class for synchronizing process variables on the device side.

This class makes it easier to process changes coming from the control system by checking the notification queue and only receiving those process variables which have new values pending.

Register a receive notification listener for a process variable.

This listener is triggered every time after calling receive on the process variable if the receive operation returned true.

If another listener has previously been registered for the named process variable, that listener is replaced with the new listener.

void ChimeraTK::DeviceSynchronizationUtility::sendAll ( )

Sends all process variables that are senders.

This method cannot know which process variables have actually changed, therefore one of the send(...) methods that take a list of process variables should be preferred. However, this method can be used if the the device library always updates all process variables that transfer data to the control system.

◆ waitForNotifications()

void ChimeraTK::DeviceSynchronizationUtility::waitForNotifications ( long  timeoutInMicroseconds,
long  checkIntervalInMicroseconds 
)

Periodically runs receiveAll() in order to process incoming changes.

This method calls receiveAll() every checkIntervalInMicroseconds microseconds, until timeoutInMicroseconds microseconds have passed in total, sleeping in between. If either of the parameters is less than or equal to zero, this method returns immediately after calling receiveAll() once.

This is useful if you want to spend time sleeping while still periodically checking for new values.
