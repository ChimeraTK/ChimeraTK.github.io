const std::string &  name,
const std::string &  description,
bool  eliminateHierarchy = false,
const std::unordered_set< std::string > &  tags = {} 
)

Constructor: register the module with its owner.

If eliminateHierarchy is true, the hierarchy level introduced by this module will be eliminated from the "dynamic" data model (see EntityOwner::setEliminateHierarchy()). The tags given as the last argument are added to all variables in this module recursively (see EntityOwner::addTag()).

Note: ApplicationModules may only be owned by ModuleGroups or Applications.

ChimeraTK::ApplicationModule::ApplicationModule ( )
inline

Default constructor: Allows late initialisation of modules (e.g.

when creating arrays of modules).

This construtor also has to be here to mitigate a bug in gcc. It is needed to allow constructor inheritance of modules owning other modules. This constructor will not actually be called then. See this bug report: https://gcc.gnu.org/bugzilla/show_bug.cgi?id=67054
