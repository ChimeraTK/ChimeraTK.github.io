Detailed Description

template<class derived>
class mtca4u::VirtualLab::VirtualLabBackend< derived >

Base class for VirtualLab dummy devices.

The VirtualLabBackend is an extension of the DummyBackend. Like its base class, it implements a set of registers defined in a map file in memory. VirtualLabBackend helps implementing the functionality, which will be provided by the firmware and hardware in case of a real device. The implementation of firmware functionality is realised using a state machine. Inputs and outputs of the simulated device can be added using the SignalSink and SignalSource classes, which can then be connected with other VirtualLab components in the virtual lab setup code.

Use the following macros to build your VirtualLabBackend:

  • CONSTRUCTOR to create the constructor including parts needed for registration with the BackendFactory
  • REGISTER_BACKEND_TYPE needs to be placed in the C++ file to register with the BackendFactory
  • DECLARE_REGISTER and DECLARE_MUXED_REGISTER to obtain register accessors for registers
  • DECLARE_MAIN_STATE_MACHINE to declare your main state machine with transition table
  • DECLARE_STATE_MACHINE to declare any sub state machine
  • DECLARE_STATE, DECLARE_EVENT, DECLARE_GUARD and DECLARE_ACTION to build state machine components
  • DECLARE_LOGGING_STATE to create states with debug output to the console
  • DECLARE_REGISTER_GUARD to declare a guard condition using a register
  • WRITEEVENT_TABLE, READEVENT_TABLE and CONNECT_REGISTER_EVENT to create events firing on reading or writing registers
  • DECLARE_TIMER and DECLARE_TIMER_GROUP to create timers and put them into a group
  • INIT_SUB_STATE_MACHINE inside the CONSTRUCTOR to initialise a sub state machine created with DECLARE_STATE_MACHINE

Note: The bare minimum usually is one DECLARE_STATE_MACHINE with one DECLARE_STATE, plus one DECLARE_TIMER_GROUP containing one timer.
