StepperMotorRet ChimeraTK::motordriver::StepperMotorWithReference::calibrate ( )
virtual

Calibration of the linear stage.

This initiates a drive through the full range between both end switches and determines their position.
The negative end switch will be defined as zero. The resulting coordinates can later be translated by the translateAxis() routines.
On success, this function will result in the calibration mode being StepperMotorCalibrationMode::FULL.

StepperMotorRet ChimeraTK::motordriver::StepperMotorWithReference::determineTolerance ( )
virtual

Determines the standard deviation of the end switch position.

The standard deviation is determined by running into each end switch multiple times.
If this has been called on system setup, getError() will return StepperMotorError::CALIBRATION_LOST
error when an end switch is activated and the actual position is not within a 3 sigma band around the
calibrated end switch position.

StepperMotorError ChimeraTK::motordriver::StepperMotorWithReference::getError ( )
virtual

Return error code for the motor.

The error code is not a bit-field.
0 - NO_ERROR
1 - ACTION_ERROR if any of the action started by the user was not successful
2 - MOVE_INTERUPTED if the motion of the motor was interrupted, e.g. a software/hardware limit was hit.
3 - CALIBRATION_LOST if calibration is lost.

StepperMotorRet ChimeraTK::motordriver::StepperMotorWithReference::setActualPositionInSteps ( int  actualPositionInSteps)
virtual

Simple calibration of the linear stage by defining the actual position in steps.

This function only sets the actual position to a defined value. The end switch positions will not be defined.
This is useful in applications where the stage can not move through the full range between the end switches.
This function will result in the calibration mode being StepperMotorCalibrationMode::SIMPLE.

Translate axis of the linear stage by offset translationInSteps.

The actual position and calibrated values of the end switch positions are shifted by the given offset. The resulting new position will be truncated if the calculated value exceeds the numeric limits of an int.
