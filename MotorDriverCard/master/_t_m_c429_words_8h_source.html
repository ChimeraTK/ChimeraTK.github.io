7 // All bits from the ADD_VARIABLE are only in this file. Their correctness is tested
8 // with a unit test (second, independent implementation of the same information).
9 // I intentionally decided to directly write the numbers in the code and not define
10 // constants for each start and stop bit. These constants whould only tempt users
11 // to use them in their code and start bit-fiddeling themselves. In addition, having a
12 // header with so many constants would not make it more "storing them all in one place" than
13 // having the numbers here, within their context and two numbers, which belong together, per line.
