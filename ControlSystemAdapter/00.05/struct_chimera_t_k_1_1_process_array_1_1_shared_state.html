template<class T>
boost::lockfree::spsc_queue< boost::shared_future<void>, boost::lockfree::capacity<2> > ChimeraTK::ProcessArray< T >::SharedState::_notificationQueue

Queue holding futures for notification.

The receiver can obtain a future from this queue when the _fullBufferQueue is empty and wait on it until new data has been sent.

It is sufficient to have a fixed queue length of 2, since it is only important to guarantee the presence of at least one unfulfilled future in the queue.

We are using a shared future, since the spsc_queue expects a copy-constructable data type. Since the data type void is not particularly expensive to copy, this should have no performance impact.
